# Chapter 1 운영체제 개요

## 1 운영체제 개념

#### 1.1 운영체제 정의
컴퓨터 자원을 효율적으로 관리하고  
다양한 응용프로그램이 원할하게 작동할 수 있도록 지원하는  
소프트워어

- 컴퓨터는 하드웨어와 소프트웨어로 구성된다.
- 소프트웨어는 용도에 따라 응용 프로그램, 시스템 소프트웨어로 나뉜다.
  - 응용 프로그램: 특정한 목적을 위해 사용되는 소프트웨어 (ex, MS워드, 아래한글)
  - 시스템 소프트웨어(운영체제): 하드웨어, 응용 프로그램을 관리하는 소프트웨어

#### 1.2 운영체제의 역할

1. 자원 관리 -> 컴퓨터 자원을 여러 프로그램이 동시에 사용
2. 자원 보호 -> 무분별한 접근을 막아 보안성, 안정성 확보
3. 소프트웨어 인터페이스 -> 사용자/응용프로그램이 복잡한 하드웨어 구조를 몰라도 자원을 쉽게 사용 가능
4. 하드웨어 인터페이스 -> 다양한 주변장치와 컴퓨터의 원활한 소통

## 3 운영체제 구성

운영체제의 두가지 핵심 구성 요소
1. 사용자 인터페이스 - 사용자, OS 간 소통
2. 커널 - OS의 핵심 부분

#### 3.1 사용자 인터페이스

- **의미**: 컴퓨터 자원에 접근 가능한 커널이 제공하는 인터페이스
- **목적**: 사용자가 컴퓨터를 이해/제어할 수 있도록 돕는다
- **종류**: CLI, GUI

#### 3.2 커널

- 의미: OS의 핵심기능을 수행하는 중앙 제어 소프트웨어
- 목적: 프로세스, 메모리, 저장장치, 입출력 관리의 네가지 기능 -> 응용프로그램 실행 조율, 하드웨어 자원 효율적 관리

##### 3.2.1 시스템 호출
- **의미**: 응용 프로그램이 커널에게 보내는 요청
- **목적**:
  - 자원 보호: 응용 프로그램이 직접 자원에 접근하여 자원의 무분별 사용 방지
  - 접근 권한 통제: 접근 권한을 통제

##### 3.2.2 이중 모드 연산

사용자 모드와 커널 모드를 오가는 작업 방식

- 일반 프로세스가 자발적으로 커널 모드에 진입할 수 있는 유일한 방식은 시스템 호출 뿐이다

> #### 저수준 함수와 고수준 함수
> **저수준 함수**:
> - 하드웨어나 운영체제의 자원에 직접 연결되어 커널 모드에서 실행된다
> - 파일 descriptor를 직접 다루고, 별도의 버퍼링을 제공하지 않는다
> - 속도는 빠르지만 사용법이 비교적 복잡, 프로그래머의 세부 동작이 필요
> 
> **고수준 함수**:
> - 표준 C 라이브러리 같은 것을 말함
> - 저수준 함수를 사용하여 구현된다
> - 내부적으로 버퍼링, 입출력 효율을 향상, 오류 처리 자동

##### 3.2.3 디바이스 드라이버

- **의미**: 커널과 하드웨어 사이의 하드웨어 인터페이스
- **목적**: 커널과 주변 장치의 데이터 교환

- 커널 - 주변장치의 표준 통신 방식은 기본, 필수적인 기능만을 포함한다
- 일부 고급 기능, 성능 향상을 위한 추가 기능은 장치 제작자가 별도로 디바이스 드라이버에 포함하여 제작한다
  -> 이 경우, 사용자가 해당 디바이스 드라이버 소프트웨어 설치 필요
  (ex, 그래픽카드의 고성능 연산 기능, 프린터의 다양한 출력 옵션)

  > #### 응용프로그램 인터페이스와 시스템 개발자 키트
  > **응용프로그램 개발자 키드 (API)**
  > - 특정 플랫폼, 소프트웨어 관련된 프로그램을 개발할 수 있도록 제공되는 인터페이스
  > - 예시: 구글맵 API
  >
  > **시스템 개발자 키드 (SDK)**
  > - API + 각종 개발 도구(API 사용 매뉴얼, 코드 편집기, 에뮬레이터, ..)를 묶은 종합 개발 도구
  > - 예시: 안드로이드 스튜디오 

### 3.3 커널 구조

##### 3.3.1 단일형 구조

모든 커널 기능 -> 하나의 큰 모듈로 구현

- **특징**: 
  - 초기 구조
  - 모든 기능이 단일 커널 공간에서 실행
  - 각 기능은 커널 내부에서 밀접하게 통합됨
  - 과거에는 운영체제의 크기가 크지 않아 문제가 단일형 구조로 만들 수 있었다
 
- **장점**:
  - 구현이 간단하다
  - 모듈이 분리되지않아 모듈간 통신 비용이 적다
    
- **단점**:
  - 상호의존성이 높아 버그나 오류를 처리하기 어렵다
  - 이식성이 낮다

##### 3.3.2 계층형 구조

비슷한 기능을 가진 모듈을 묶어 하나의 계충으로 만들어 커널을 구현한다

- **특징**: 
  - 상위 계층은 사용자 인터페이스와 관련된 작업을 처리
  - 각 계층이 명확한 역할을 가지면서 계층 간 인터페이스를 통해 통신한다
  - 오늘날의 대부분의 운영체제의 구조

- **장점**:
  - 오류 발생 시 해당 계층만 따로 수정-> 유지, 관리가 쉽다
    
- **단점**:
  - 모듈간 통신 오버헤드가 증가한다

##### 3.3.3 마이크로 구조

커널에는 최소한의 기능만 두고, 나머지는 사용자 영역에 구현한다

- **장점**:
  - 안정적이다 -> 각 모듈은 독립적으로 작동하기 때문에 모듈 1개에 이상이 생겨도 전체 운영체제가 멈추지 않는다
  - 이식성이 좋다
  - 커널이 가벼워 CPU 용량이 작은 시스템에도 적용 가능하다

> ##### 하이브리드 커널
> 단일형 구조 + 마이크로 구조의 장점을 결합한 커널
> - 주요 기능은 커널 내부에서 실행, 일부 기능은 모듈 형태로 분리
> - 단일형 구조 커널과 유사하다고 보이기도 한다.
