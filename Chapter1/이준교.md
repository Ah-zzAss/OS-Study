# Chapter 01. 운영체제 개요

## 1. 운영체제의 개념

### 1.1. 운영체제의 정의

컴퓨터는 `하드웨어`와 `소프트웨어`로 구성되며, 유기적으로 결합되어 다양한 작업을 수행한다.

#### 소프트웨어

- 응용 프로그램
    - 특정한 목적을 위해 사용되는 소프트웨어
    - 아래아한글, MS워드
- 시스템 소프트웨어
    - 하드웨어와 응용 프로그램을 관리하는 소프트웨어
    - 운영체제

### 1.2. 운영체제의 역할과 목표

`운영체제`는 컴퓨터에서 관리자 역할을 수행하며, 시스템 자원을 안전하고 효율적으로 관리하여 전체 컴퓨터 시스템이 원활하게 작동하도록 돕는다. 또, 사용자가 하드웨어를 직접 다루지 않고도 다양한 작업을 수행할 수 있도록 중간에서 제어와 조율을 담당한다.

#### 운영체제의 역할

- 자원 관리 기능
    - CPU, 메모리, 저장장치, 입출력 장치와 같은 컴퓨터 자원을 여러 프로그램이 동시에 사용할 수 있도록 적절히 배분
- 자원 보호 기능
    - 프로그램이나 사용자가 시스템 자원에 무분별하게 접근하지 못하도록 제한함으로써, 보안성과 안정성을 확보
- 소프트웨어 인터페이스 제공
    - 사용자나 응용 프로그램이 복잡한 하드웨어 구조를 몰라도 자원을 쉽게 활용할 수 있도록 함
- 하드웨어 인터페이스 제공
    - 다양한 주변장치들이 컴퓨터와 원활히 연결되도록 도움

#### 운영체제의 목표

- 효율성
    - 같은 자원으로 더 많은 작업을 처리
    - 더 적은 자원으로 동일한 작업을 수행
- 안정성
    - 시스템이 예기치 않은 오류나 외부 충격으로부터 쉽게 중단되지 않도록 보호
    - 문제가 발생해도 빠르게 복구 및 작업 중 데이터가 손실되지 않도록 함
- 편리성
    - 사용자나 프로그램이 시스템 자원에 쉽게 접근하고 사용할 수 있도록 함
- 확장성
    - 새로운 하드웨어나 소프트웨어가 추가될 때도 시스템이 이를 유연하게 받아들이고, 다양한 장치와의 호환성을 유지할 수 있도록 함

## 3. 운영체제의 구성

### 3.1. 사용자 인터페이스

`사용자 인터페이스`는 사용자가 컴퓨터를 이해하고 제어할 수 있도록 돕는 소프트웨어적 인터페이스이다.

- CLI(문자 기반 인터페이스)
    - 사용자가 명령어를 직접 입력하여 작업을 지시
- GUI(그래픽 사용자 인터페이스)
    - 마우스와 아이콘을 이용하여 직관적인 방식으로 컴퓨터 조작 가능

### 3.2. 커널

#### 커널의 핵심 기능

`커널`은 운영체제의 핵심 기능을 수행하는 중앙 제어 소프트웨어이다.

- 프로세스 관리
    - 프로세스에 CPU를 배분하고 작업 환경을 제공
- 메모리 관리
    - 메인 메모리에 프로세스 배치
- 저장장치 관리
    - 저장장치에 파일 시스템을 생성하고 관리
- 입출력 관리
    - 주변장치의 입출력을 관리

#### 시스템 호출

`시스템 호출`은 커널이 제공하는 시스템 함수를 호출해서 사용하는 것이다.

응용 프로그램은 직접 자원에 접근하지 않고, 커널이 제공하는 시스템 함수를 통해 요청을 보낸다. 커널은 요청을 받아서 자원을 안전하게 처리한 후, 그 결과만 사용자에게 전달한다.

#### 이중 모드 연산

프로세스는 기본적으로 사용자 모드에서 실행된다. 자원에 접근해야 하는 작업은 시스템 호출을 통해 커널 모드로 전환되어야 한다.

`이중 모드 연산`은 사용자 모드와 커널 모드를 오가는 작업 방식을 말한다.

사용자 모드에서는 커널 자원에 직접 접근할 수 없고, 시스템 호출을 통해서만 커널 함수에 접근할 수 있다. 따라서 일반 프로세스가 자발적으로 커널 모드에 진입할 수 있는 유일한 방법은 시스템 호출뿐이다.

#### 디바이스 드라이버

`디바이스 드라이버`는 커널과 주변장치가 데이터를 주고받기 위해 사용하는 하드웨어 인터페이스이다.

### 3.3. 커널 구조

#### 단일형 구조

`단일형 구조` 커널은 모든 커널 기능을 하나의 큰 모듈로 구현한 커널이다.

- 장점
    - 구현이 간단함
        - 모듈 간 통신 비용이 적어 효율적으로 운영 가능
- 단점
    - 버그나 오류 처리가 어려움
        - 운영체제의 여러 기능이 서로 연결되어 있어 상호 의존성이 높음 -> 기능상의 작은 결함이 시스템 전체로 확산 가능
    - 이식성이 낮음
        - 모듈화가 되어 있지 않기 때문에 특정 부분을 찾아 수정하기 어려움


#### 계층형 구조

`계층형 구조` 커널은 단일형 구조 커널이 발전된 형태이다. 비슷한 기능을 가진 모듈을 묶어서 하나의 계층으로 만들어 커널을 구현한다.

- 장점
    - 기능에 따라 모듈화했기 때문에 단일형 구조보다 오류를 쉽게 처리 가능
- 단점
    - 단일형 구조와 비교해 모듈 간의 통신 오버헤드 증가

#### 마이크로 구조

`마이크로 구조` 커널의 운영체제는 프로세스 관리, 메모리 관리, 프로세스 간 통신 관리 등 가장 기본적인 기능만 제공하고, 운영체제의 많은 부분이 사용자 영역에 구현되어 있다.

- 장점
    - 각 모듈이 독립적으로 작동하기 때문에 모듈 1개에 이상이 생겨도 전체 운영체제가 멈추지 않아 안정적
    - 커널을 어떤 컴퓨터 환경에든 이식하기 쉽고, 커널이 가벼워서 CPU 용량이 작은 시스템에도 적용 가능
